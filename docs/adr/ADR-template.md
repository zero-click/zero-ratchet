# ADR Template

## ADR-XXXX: <Decision Title>

### 1. Status
- Proposed | Accepted | Rejected | Superseded

### 2. Context
- What problem are we solving?
- Which domains are affected? (UI / Backend / Database / Infra)
- What constraints are confirmed by user/business/compliance?

### 3. Baseline Decision
- Baseline selected per affected domain
- Why this is mainstream, maintainable, and evolvable for this project

### 4. Deviation (if any)
- Deviation detected: true|false
- Exact deviation description
- Why baseline is not suitable here
- Risk and cost impact

### 5. Options Considered
- Option A (baseline)
- Option B (deviation)
- Option C (alternative)
- Tradeoffs summary

### 6. Decision
- Chosen option
- Approval reference (`approval_ref`)
- Effective date / scope

### 7. Consequences
- Positive impacts
- Negative impacts
- Follow-up tasks / rollback trigger

### 8. Verification
- How design-review/code-review gates will verify compliance
- Evidence paths (tests, docs, config)
